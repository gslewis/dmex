Tables Revision

Overview

Tables Revision produces 2-term multiplication and division problems in a range of formats of varying difficulty.

Term Selection

To define a problem it is necessary to select the required values for term1 and for term2. (Note that term1 does not necessarily appear on the left side of the equation.) Each term can include any set of integers from 1 to 12 or All values (ie., 1 to 12 inclusive).

Difficulty

It is also necessary to select the problem difficulty which controls the range of problem types that are generated. There are 16 problem types. At the Easiest difficulty level, only the first two types are used. At the Hardest difficulty level, all 16 types are used. The problem types are:

  1. Standard multiplication with answer in right-hand side (RHS)
  2. Same as Type 1 but swapping term1 and term2
  3. Standard multiplication with answer in left-hand side (LHS) second term
  4. Same as Type 3 but swapping term1 and term2
  5. Standard multiplication with answer in LHS first term
  6. Same as Type 5 but swapping term1 and term2
  7. Standard division with answer in RHS
  8. Standard division with answer in LHS second term
  9. Standard division with answer in LHS first term
  10. Fractional multiplication with answer in RHS, fraction in LHS second term
  11. Fractional multiplication with answer in RHS, fraction in LHS first term
  12. Fractional multiplication with answer in LHS first term, fraction in LHS second term
  13. Fractional multiplication with fraction in LHS second term and the answer in the denominator
  14. Fractional division with fraction in LHS second term and answer in RHS.
  15. Fractional division with fraction in LHS second term and answer in LHS first term.
  16. Fractional division with fraction in LHS second term and answer in denominator.

The number of possible problems is further expanded by flipping the equation about the "=" sign.

Depending on the problem definition that is chosen (ie., the possible values of term1 and term2 as well as the difficulty level), a certain number of unique problems can be created. For example, if only one number is picked for each of term1 and term2 on the Easiest difficulty level, then there are only 4 unique problems.
